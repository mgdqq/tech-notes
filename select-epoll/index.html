select 与 epoll 工作模式

动态观察一个线程如何等待多个 socket：select 每次被唤醒后扫描整组 fd；epoll 由内核维护关注集合，并把已就绪 fd 放入就绪队列。

未就绪 fd 数据已就绪 select 扫描 epoll 就绪队列 应用处理 read/recv

select 轮询扫描

线程阻塞在 select 调用上；任意 fd 就绪后返回，应用层需要遍历传入的 fd 集合，逐个检查并处理已就绪项。

内核等待
0唤醒次数
0fd 检查次数

epoll 就绪队列

线程阻塞在 epoll_wait；fd 就绪时内核把事件放入 ready list，返回时应用通常只遍历就绪事件，不必每轮扫描全部 fd。

内核等待
ready list
0唤醒次数
0返回事件数
0处理事件数

重点：select 和 epoll 都属于 IO 多路复用。它们可以让一个线程等待多个 fd，但等待期间这个线程仍然阻塞在等待调用中；区别主要在“就绪之后如何发现哪些 fd 可读/可写”。
